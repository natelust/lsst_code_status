commit 81fe29036239e96d5cf4bb7b2208dcac47a69803
Author: Paul Price 
Date:   Thu May 21 15:43:11 2015 -0400

    add measure of blendedness
    
    The blendedness of a source is defined as:
        [heavy[i].dot(heavy[j]) for j in neighbors].sum() / heavy[i].dot(heavy[i])
    This is a dimensionless quantity, compared to the originally proposed
        [heavy[i].dot(heavy[j]) for j in neighbors].sum() / heavy[i].sum()
    which has dimensions of flux --- but note that just dividing this quantity by
    the total flux may not isn't useful, because the numerator is a sum of squares;
    that's why I decided to try using the dot product in the denominator instead.

    Registry: hold registry cache for a single thread only (HSC-1035)
    
    We destroy the connection cache when we discover that we're in
    a new thread, to avoid using a SQLite connection from a thread
    other than which it was created in (resulting in: "ProgrammingError:
    SQLite objects created in a thread can only be used in that same
    thread.").  multiprocessing runs a thread within each process
    (only one at a time), and different iterations in a process appear
    to use different threads.
    
    This is not attempting to be thread-safe (this is inherently NOT
    thread-safe --- there is no locking or thread-specific data), but
    to support multiprocessing, as SQLite (and possibly others) will not
    use a connection created in a different thread.
    
    Could have chosen to hold the cache of connections indexed on the
    thread identifier, but that means several connections for the same
    database could be open at the same time, which defeats the purpose
    of the cache (or, at least, makes it less effective).

    reimplement color term persistence as Config instead of global
    
    A global being set in a config override file does not pickle, since
    it's relying on a side-effect.  Therefore, in order for color term
    selection to be picklable, it needs to be in a Config.  This new
    design uses two layers of ConfigDictField to create a menu for
    selecting the astrometry_net_data version and target band of
    interest.  The bottom-level ColortermConfig can construct a
    Colorterm instance.
    
    Because the full list of colorterms is no longer globally accessible,
    the Astrometry needs to be told whether to pull all available fluxes
    (it can no longer dig into some other module's private information);
    it's best to default this to 'True' to ensure the user gets what he
    needs.
